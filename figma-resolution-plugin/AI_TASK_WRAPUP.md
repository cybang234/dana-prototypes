# AI 과업 랩업 — Figma Resolution Converter 플러그인 제작기

> **한 줄 요약:** 코드를 모르는 Product Designer가 Claude Code(AI)와 함께 Figma 플러그인을 처음부터 끝까지 만들었다.

---

## 배경

Pay 팀에서 디자인할 때 반복되는 작업이 있었다.

모바일(390px)로 완성한 스크린을 PC(1440px), Tablet(768px), Wide(1920px)로 옮겨야 할 때마다 **직접 레이아웃을 손으로 다시 잡아야 했다.** Status Bar 삭제, Nav 교체, 콘텐츠 너비 대응, 상품 그리드 컬럼 재배치… 스크린 하나당 20~30분.

이 반복 작업을 자동화할 수 있을지 AI에게 물어봤다.

---

## 무엇을 만들었나

**Resolution Converter** — Figma 플러그인

모바일 프레임을 선택하고 해상도 프리셋을 고르면 자동으로 변환해주는 플러그인이다.

### 주요 기능

| 기능 | 설명 |
|------|------|
| 해상도 프리셋 | Mobile(375), Tablet(768), PC(1440), Wide(1920) 복수 선택 |
| Responsive 모드 | Status Bar 제거, 글로벌 Nav 삽입, 콘텐츠 width 대응, 상품 그리드 자동 변환 |
| Fixed 모드 | 콘텐츠 최대 568px 중앙 정렬 (결제/팝업 스크린용) |
| Nav 모드 | 모바일 Nav 유지(keep) / 웹 Nav로 교체(replace) 선택 |
| 자동 CTA 고정 | `_auto_spacer` 프레임으로 CTA를 하단에 자동 고정 |

---

## AI와 어떻게 일했나

### 도구

- **Claude Code** (Claude Sonnet 4.6) — Figma Plugin API 코드 작성, 디버깅, 로직 설계 전부

### 작업 방식

코드는 전혀 직접 쓰지 않았다. 대신 **Figma에서 발견한 문제를 링크와 함께 Claude에게 보고** → Claude가 코드를 수정하는 루프를 반복했다.

```
[내가 한 일]
Figma에서 변환 결과 확인
→ 문제 발견 (스크린 링크 첨부)
→ "이 컴포넌트 width가 안 맞아" / "홈인디케이터 마진이 생겨"
→ Claude가 코드 분석 후 수정
→ 플러그인 재로드 후 재확인
```

총 **세션 여러 개**에 걸쳐 약 **20개 이상의 버그/개선**을 이 방식으로 처리했다.

## 주요 기술 챌린지

코드를 몰라도 어떤 문제가 있었는지는 알 수 있었다. 버그 리포트를 구체적으로 할수록 Claude의 수정이 정확해졌다.

### 1. 너비 대응이 안 되는 컴포넌트

**현상:** 일부 자식 프레임이 변환 후에도 390px(모바일 너비)로 고정됨

**원인:** 재귀 함수의 임계값이 비율 기반(`oldWidth × 0.6`)이었는데, 이미 한 번 변환된 outer 프레임(768px) 안에 있는 inner 프레임(390px)이 조건을 통과 못 함

**해결:** 절대값 기준(`width > 100px`)으로 변경 — 아이콘/소형 요소는 건드리지 않고 콘텐츠 프레임만 대응

---

### 2. HUG 텍스트가 FILL로 변환됨

**현상:** 텍스트 너비가 콘텐츠에 맞게 HUG되어야 하는데 FILL로 늘어남

**원인:** width > 60px인 텍스트도 일괄 FILL 처리되는 로직

**해결:** Figma API의 `textAutoResize === 'WIDTH_AND_HEIGHT'` 속성으로 HUG 텍스트 감지 후 예외 처리

---

### 3. 닫기 버튼이 삭제됨

**현상:** PC Fixed 모드에서 Top Nav의 닫기 버튼(X)이 사라짐

**원인:** Nav 처리 로직에서 Left Icons / Right Icons 전체를 제거하는 코드가 포함되어 있었음

**해결:** 해당 코드 완전 제거 — 아이콘 그룹은 그대로 유지하고 타이틀 프레임만 FILL 처리

---

### 4. _auto_spacer가 사라짐

**현상:** CTA를 하단에 고정하기 위해 삽입한 `_auto_spacer` 프레임이 변환 후 사라지거나 height 0이 됨

**원인:** 내부 자식 처리 재귀 함수가 spacer도 동일하게 처리

**해결:** spacer 이름 감지 후 `continue` 처리로 스킵. Fixed 모드에서도 spacer가 없으면 CTA 앞에 자동 삽입하는 로직 추가

---

### 5. keep 모드 홈인디케이터 하단 마진

**현상:** 모바일 Nav 유지(keep) 모드에서 홈인디케이터 아래에 불필요한 마진 발생

**원인:** BOTTOM_PADDING이 keep 모드 조건 없이 전체 적용됨

**해결:** `navMode !== 'keep'` 조건 추가. 단, 원본 프레임에 홈인디케이터가 없는 경우엔 20px 마진 자동 추가

## 배운 것

### AI에게 일 잘 시키는 법

1. **Figma 링크를 첨부한다** — "이거 이상해"보다 "노드 183:134431 보면 Left Icons가 FILL로 늘어남"이 훨씬 빠르다
2. **현상과 기대값을 함께 말한다** — "왜 이렇게 되는지"보다 "이렇게 돼야 하는데 이렇게 됨"
3. **하나씩 리포트한다** — 여러 버그를 한 번에 던지면 수정이 뒤섞인다

### 코드 없이 플러그인을 만든다는 것

Figma Plugin API는 복잡하다. `layoutSizingHorizontal`, `primaryAxisSizingMode`, `textAutoResize`… 수십 개의 속성을 맥락에 맞게 조합해야 한다.

직접 익히려면 몇 주가 필요했을 지식을 AI가 대신 보유했고, 나는 **무엇이 문제인지 정확히 관찰하고 전달하는 역할**을 했다.

결국 이 플러그인은 **디자이너의 관찰력 + AI의 구현력**이 합쳐진 결과물이다.

- 해상도 변환 작업: 스크린 당 **20~30분 → 10초**
