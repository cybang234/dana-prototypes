# Dana Todo — Data Playground 빌드 요청서

> 이 문서를 사내 Claude(슬랙·사내 웹)에 그대로 붙여넣고
> 마지막에 `/dataplay` 호출 + "위 spec대로 만들어줘"라고 요청하면 빌드 시작됩니다.

---

## 한 줄 정의

슬랙 메시지에 `danachecking` 이모지를 달면, 그 메시지가 한 채널에 모이고,
AI가 우선순위 매겨서 모바일 todo 앱·위젯에 자동으로 띄워주는 개인 업무 자동화 도구.

(토스 기술블로그 "매일 하던 업무를 디자인하기" 글의 todolist 패턴 동일)

## 데이터 흐름

1. **수집**: 다른 채널 메시지에 `:danachecking:` 이모지 단다
2. **포워딩** (이미 완료): 슬랙 워크플로우 `danatodo`가 메시지를 `#todo-list` 채널(ID `C0BEMBM8M16`)에 자동 포스팅
3. **읽기**: 시스템이 `#todo-list` 채널을 주기적으로 polling (5~15분)
4. **분류**: LLM이 메시지 본문을 보고 다음 자동 분류
   - **우선순위** P1(긴급)/P2(이번주)/P3(여유)/P4(낮음)
   - **종류** 버그/요청/피드백/결정/질문/협의/기타
   - **부서** 영업/개발/디자인/MD/CS/마케팅/페이/PD/기타
   - **한 줄 제목** 메시지 본문 요약
5. **저장**: 사내 DB에 todo 레코드 누적
6. **출력**: 모바일 웹앱(PWA) + 알림센터 위젯

## 알림 정책

- 새 todo 추가 시: 푸시 알림 (제목 + 우선순위)
- 우선순위 P1 항목 마감 30분 전: 푸시 알림
- 매일 아침 9시: 오늘 P1·P2 요약 알림

---

## 자동화 정책

- Slack `#todo-list` 채널 polling: **15분 간격**
- 신규 메시지 감지 시 즉시 LLM 분류 → DB 저장
- 완료 처리: 원본 메시지에 `danachecked` 이모지 달리면 자동 done 상태로 전환
- Google Calendar 연동: 오늘 일정 fetch (매 30분)

## DB 스키마

**Table: todos**

| 컬럼 | 타입 | 비고 |
|---|---|---|
| id | string | slack ts 기반 unique |
| title | string | LLM 요약 (한 줄) |
| body | text | 원본 메시지 본문 |
| priority | enum | P1/P2/P3/P4 |
| category | enum | 버그/요청/피드백/결정/질문/협의/기타 |
| team | enum | 영업/개발/디자인/MD/CS/마케팅/페이/PD/기타 |
| requester | string | 메시지 작성자 이름 |
| source_url | string | 슬랙 permalink |
| channel | string | 출처 채널명 |
| created_at | timestamp | 이모지 단 시각 |
| completed_at | timestamp | nullable, danachecked 시각 |
| status | enum | checking/done |

## 우선순위 분류 기준 (LLM 프롬프트 가이드)

LLM에게 todo 분류 시 다음 기준 적용:

**P1 (긴급, 오늘 안)**
- "지금", "긴급", "asap", "오늘까지" 같은 단어 포함
- 버그·장애 신고
- 시간대 명시된 외부 일정

**P2 (이번 주 안)**
- "이번 주", "주중에", "곧" 같은 단어
- 디자인 시안 리뷰·결정
- 협업 요청 (마감일 모호)

**P3 (다음 주 이후)**
- "다음에", "여유 될 때", "참고용"
- 단순 공유·질문
- 백로그성 요청

**P4 (낮음)**
- 명확한 마감일 없는 정보성
- 시스템 알림·뉴스레터
